# Briefs por líder de sección

Un brief por cada líder contratado. Regla de diseño de estos documentos:

- **Visibilidad total de la arquitectura:** todo brief abre con el mismo mapa del proyecto
  completo, para que cada líder entienda el sistema entero y dónde encaja su trabajo.
- **Alcance acotado:** el detalle operativo (specs, decisiones internas, entregables finos)
  solo aparece en la sección propia. Lo de las demás secciones se describe al nivel de
  "qué recibes de ellos / qué les entregas", nada más.
- **Cero términos comerciales:** tarifas, montos y condiciones de pago viven únicamente en el
  contrato individual de cada persona, nunca en estos archivos (el repo es público).
